Piecewsise functions - Library usage¶

This notebook is a simple walkthrough of this library and what you can do with it

Features¶

The source code is located under src/PiecewiseFunctions. You'll find two piecewise functions implemented:

  1. A piecewise constant function, PiecewiseConstantFunction Defined by the equation $y = b$ over a domain $D = [c,d]$ where $(b,c,d) \in \mathbb{R}^3$, where we have multiple definition domain $D$ (defined by the breakpoints in the proposed implementations)

  2. A piecewise linear function, PiecewiseLinearFunction Defined by the equation $y = a*x + b$ over a domain $D = [c,d]$ where $(a,b,c,d) \in \mathbb{R}^4$, where we have multiple definition domain $D$ (defined by the breakpoints in the proposed implementations)

Both inherit from the PiecewiseFunction abstract class.

Every implementation of a piecewise function implements the 3 following methods:

  • Evaluate
def evaluate(self, x: float) -> float:
    """ Return the value of the function on the given value if defined, otherwise raise a ValueError """
    pass
  • Maximum
def maximum(self) -> float:
    """ Return the maximum value of the function over its definition domain """
    pass
  • Minimum
def maximum(self) -> float:
    """ Return the minimum value of the function over its definition domain """
    pass

piecewise_fn = PiecewiseConstantFunction(
        breakpoints=[-100.00, -50, 10, 20, 31.0],
        values=[30, 20, 10, -10]
    )
draw(
    piecewise_fn,
    x_min=-100,
    x_max=30,
    num_points=1000,
    title="Piecewise Constant Function",
)

The function call wan be understood this way: We have a function whose value is defined as so: $$

f(x) = \left{ \begin{array}{ll} 30 \mbox{ if } x \mbox{ \in } [-100, -50[ \\ 20 \mbox{ if } x \mbox{ \in } [-50, -10[ \\ 10 \mbox{ if } x \mbox{ \in } [-10, 20[ \\ -10 \mbox{ if } x \mbox{ \in } [20, 31[ \\ \end{array} \right. $$

The domains of definitions are pairs of borders in the breakpoints list [-100.00, -50, 10, 20, 31.0], and the constants over those domaines are in the values list [30, 20, 10, -10].

Note that the function is not defined in 31.

Minimum & maximum¶

To request the min, (resp. max) value and the $arg_{min}$ (resp. $arg_{max}$) value, use the minimum() method (resp. maximum()).

In [3]:
min_value, arg_min = piecewise_fn.minimum()
max_value, arg_max = piecewise_fn.maximum()
print(f"Min value = {min_value} obtained at arg_min = {arg_min} for the Piecewise Constant Function")
print(f"Max value = {max_value} obtained at arg_max = {arg_max} for the Piecewise Constant Function")

Min value = -10 obtained at arg_min = 20 for the Piecewise Constant Function
Max value = 30 obtained at arg_max = -100.0 for the Piecewise Constant Function

Evaluate¶

You can obtain the value of the function on any domain it is defined by running the evaluate function.

In [4]:
arg = -50
val = piecewise_fn.evaluate(arg)
print(f"Value = {val} obtained at arg = {arg} for the Piecewise Constant Function")

Value = 20 obtained at arg = -50 for the Piecewise Constant Function

PiecewiseLinearFunction¶

In [5]:
import math

breakpoints = [1, math.pi, 4, 10, 20.5, 32]
slopes = [0, -100.02, math.pi, 5, -7]
intercepts = [10, -2, 1, 3, math.pi]
piecewise_linear_fn = PiecewiseLinearFunction(breakpoints, slopes, intercepts)
draw(
    piecewise_linear_fn,
    x_min=1,
    x_max=31.9,
    num_points=10000,
    title="Piecewise Linear Function",
)

The function call wan be understood this way: We have a function whose value is defined as so: $$

f(x) = \left{ \begin{array}{ll} 10 \mbox{ if } x \mbox{ \in } [1, \pi[ \\ y = -100.02 * x -2 \mbox{ if } x \mbox{ \in } [\pi, 4[ \\ y = \pi * x +1 \mbox{ if } x \mbox{ \in } [4, 10[ \\ y = 5 * x +3 \mbox{ if } x \mbox{ \in } [10, 20.5[ \\ y = -7 * x + \pi \mbox{ if } x \mbox{ \in } [20.5, 32[ \\ \end{array} \right. $$

The domains of definitions are pairs of borders in the breakpoints list [1, math.pi, 4, 10, 20.5, 32], and the slopes over those domaines are in the slopes list [0, -100.02, math.pi, 5, -7], finally the intercepts list contains the constant [10, -2, 1, 3, math.pi].

Note that the function is not defined in 32.

Runtime evolution for PiecewiseConstantFunction

from typing import Tuple, Dict
import random
import time

def large_input_pcf() -> Dict[int, Tuple[int, int, int]]:
    """
    Returns: dict of triplet.
        For each entry:
            - the key is the number of points we consider,
            - the value (u, v, w) is a triplet of time (in seconds) to perform an evaluation, to get the max and the min

    """
    scenarios_num_points = [10, 100, 1000, 10000, 100000, 1000000, 10000000]
    res = dict()
    for num_points in scenarios_num_points:
        breakpoints = (
            [-math.inf]
            + sorted([random.uniform(-1e6, 1e6) for _ in range(num_points)])
            + [math.inf]
        )
        values = [random.uniform(-1e6, 1e6) for _ in range(num_points + 1)]

        # Create a piecewise constant function with these breakpoints and values
        pcf = PiecewiseConstantFunction(breakpoints, values)

        # Time the maximum method
        start_time = time.time()
        pcf.maximum()
        end_time = time.time()
        get_max_time = end_time - start_time

        # Time the minimum method
        start_time = time.time()
        pcf.minimum()
        end_time = time.time()
        get_min_time = end_time - start_time

        # Time the evaluate method
        start_time = time.time()
        pcf.evaluate(0)
        end_time = time.time()
        get_eval_time = end_time - start_time

        res[num_points] = (get_eval_time, get_max_time, get_min_time)

    return  res

We observe that our function is linear in its runtime complexity which is coherent with a big O analysis showing runtime of $O(n)$ for n being the number of borders in our input. The longer runtime for both get max and get min methods is expected because we do more operation of checking for conditions than our evaluation which is straightforward. If they have all the same $O(n)$ asymptotic runtime complexity, the evaluation method requires less operation in practice.
